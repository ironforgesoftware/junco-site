Places a ticket into the inbox with an atomic write — the daemon can never claim a half-written file. The filename comes from the ticket's id (falling back to the source filename); - reads the ticket from stdin, which is how other tools pipe work in. 

junco submit fix-tide-rounding.md
cat ticket.md | junco submit -
queued inbox/fix-tide-rounding.md
warn — the inbox is a code-execution boundary: a PR-flow ticket runs an agent against the repo it names, as you. Keep the inbox on a disk only you can write, and confine targets with git.allowedRepoRoots.
