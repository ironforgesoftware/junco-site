The opt-in full unification for a config still carrying legacy path keys — the ticket queue never moves automatically. Moves the queue directories into <dataDir>/queue (rename, falling back to copy+verify+delete across filesystems), runs the same in-place state-tree normalization the daemon performs at startup, then rewrites config.json to drop vaultRoot / juncoSubdir / observability.stateDir and set a non-default dataDir. Holds <dataDir>/migrate.lock so two migrations can't race; nothing already moved is ever rolled back, so fixing a conflict by hand and re-running picks up where it left off. --dry-run prints the full plan and changes nothing.

junco data migrate --dry-run
junco data migrate
warn — refuses to run while the daemon appears up: any /health response (even non-200) counts, as does a live-held worker.lock next to config.json. --force skips both checks. A state-tree name collision where the destination holds real files is left untouched and exits non-zero — file-holding data is never deleted.
